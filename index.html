系统架构

点击卡片查看各层详情，三层分离让每一层各司其职。

🏗️

framework/ 框架层

定义"怎么跑"——主流程骨架、Pipeline 调度。新增业务不碰此目录。
💼

biz/ 业务层

定义"做什么"——各 Handler 的具体搜索/排序/组包逻辑。
🧩

lib/ 算子层

定义"用什么"——可插拔的公共能力，各业务按需组合调用。
framework/
├── handler/          BaseHandler (Template Method) + HandlerManager (配置驱动注册)
├── session/          Session 基类 (KV/Any/超时) + SessionFactory (反射创建)
├── processor/        ProcessorInterface + ProcessorPipeline (YAML → 反射 → 执行)
├── server/           Server (统一请求入口 → 路由分发)
├── config/           ConfigManager (framework.yaml 解析)
├── app_context.h     全局依赖注入容器
└── class_register.h  REGISTER_MSR_HANDLER / REGISTER_MSR_PROCESSOR 宏
biz/
├── search/     全文搜索 — 多路召回 + BM25/LightGBM 粗排 + MMR 精排
├── sug/        搜索建议 — Trie 前缀匹配 + 相关性/热度/新鲜度打分
├── hint/       点后推荐 — 行为共现 + 协同过滤 + 多路召回
├── nav/        搜前引导 — 热词推荐 + 个性化排序
├── doc/        文档 CRUD API（增量索引）
└── event/      用户行为事件接入（画像更新）
lib/
├── index/       倒排索引 (BM25) + 向量索引 (HNSW/暴力) + DocStore (SQLite)
├── query/       分词 (cppjieba) + 归一化 + 同义词扩展
├── recall/      倒排召回 / 向量语义召回 / 用户历史召回 / 热门召回 / RRF 融合
├── rank/        BM25 打分 / 质量分 / 时效性衰减 / LightGBM 精排 / MMR 多样性
├── filter/      去重 / 质量过滤 / Spam 过滤 / 黑名单
├── feature/     Query 特征 / User 特征 / Doc 特征 提取
├── embedding/   bge-base-zh-v1.5 ONNX 模型 (768维) + Pseudo 降级
├── storage/     搜索词热度统计 / 行为共现存储 (SQLite)
└── user/        用户画像管理 / 事件处理 / EMA 兴趣更新

配置示例

所有行为由 YAML 配置驱动，改配置不改代码。

framework.yaml
biz/search.yaml
biz/sug.yaml
server:
  host: "0.0.0.0"
  port: 8080

embedding:
  provider: "onnx"
  dim: 768
  model_path: "models/bge-base-zh/model.onnx"
  tokenizer_path: "models/bge-base-zh/vocab.txt"

businesses:
  - business_type: "search"
    handler_name: "SearchBizHandler"
    session_name: "SearchSession"
  - business_type: "sug"
    handler_name: "SugBizHandler"
    session_name: "Session"
  - business_type: "hint"
    handler_name: "HintBizHandler"
  - business_type: "nav"
    handler_name: "NavBizHandler"

background:
  auto_train:
    enable: true
    interval_hours: 24
  auto_index_rebuild:
    enable: true
    interval_hours: 12
  sug_trie_rebuild:
    enable: true
    interval_sec: 3600
# 召回阶段 Processor 链
recall_stages:
  - name: "InvertedRecallProcessor"
    params: { max_recall: 1000 }
  - name: "VectorRecallProcessor"
    params: { enable: true, top_k: 200 }
  - name: "UserHistoryRecallProcessor"
    params: { max_recall: 50 }
  - name: "HotContentRecallProcessor"
    params: { enable: true, max_recall: 30 }

# 粗排阶段
coarse_rank_stages:
  - name: "BM25ScorerProcessor"
    params: { weight: 0.6, k1: 1.5, b: 0.75 }
  - name: "QualityScorerProcessor"
    params: { weight: 0.3 }
  - name: "FreshnessScorerProcessor"
    params: { weight: 0.1, decay_rate: 0.01 }

# 精排阶段（LightGBM 模型）
fine_rank_stages:
  - name: "LGBMScorerProcessor"
    params: { weight: 1.0 }

# 过滤阶段
filter_stages:
  - name: "DedupFilterProcessor"
  - name: "QualityFilterProcessor"
  - name: "SpamFilterProcessor"
  - name: "BlacklistFilterProcessor"

# 后处理（MMR 多样性重排）
postprocess_stages:
  - name: "MMRRerankProcessor"
    params: { lambda: 0.7, top_k: 20 }
# 搜索建议 Pipeline
recall_stages:
  - name: "TrieRecallProcessor"
    params: { max_recall: 100, prefix_match: true }

rank_stages:
  - name: "SugRelevanceProcessor"
    params: { weight: 0.5 }
  - name: "SugPostRankProcessor"
    params: { max_results: 10 }

filter_stages:
  - name: "SugBlacklistFilter"
    params: { blacklist_path: "./data/sug_blacklist.txt" }

API 接口

接口方法说明
/api/v1/searchPOST全文搜索（多路召回 + 多级排序）
/api/v1/sugPOST搜索建议（Trie 前缀匹配 + 排序）
/api/v1/hintPOST点后推荐（基于文档共现）
/api/v1/navPOST搜前引导（热词 + 个性化）
/api/v1/docPOST PUT DEL文档增删改（增量索引）
/api/v1/eventPOST用户行为事件上报
/api/v1/admin/reload_modelPOST排序模型热更新

新增业务：三步完成

零改框架代码，只需三个文件。

1️⃣

编写 Handler

继承 BaseHandler，覆写 ExtraInit / DoSearch / DoRank

文件末尾加一行：
REGISTER_MSR_HANDLER(XxxBizHandler);
2️⃣

添加 YAML 配置

config/biz/xxx.yaml — 定义 Processor Pipeline

config/framework.yaml — businesses[] 加一条路由
3️⃣

重启服务

框架自动扫描 biz/*.yaml
反射创建 Handler
注册路由

完成 ✓
