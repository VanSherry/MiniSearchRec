四大业务流程

每个业务继承 BaseHandler，由框架驱动 Pipeline 按阶段执行。核心阶段可配置、可覆写。

🔍

Search 全文搜索

多路召回 → 多级排序 → 过滤干预 → 分页输出
PreSearch — 分词 + Embedding向量化 + AB染色
Recall ×4 — 倒排(1K) + 向量(200) + 历史(50) + 热门(30) → RRF融合
CoarseRank — BM25 + Quality + Freshness → Top500
FineRank — LightGBM 精排 → Top100
Filter — 去重 / 质量 / Spam / 黑名单
Rerank — MMR多样性(λ=0.7) → Top20
Response — 分页 + JSON + 高亮 + 缓存
💡

Sug 搜索建议

输入即补全，Trie 前缀匹配 + 多维排序
CanSearch — query 非空校验
DoSearch — Trie UTF-8 前缀匹配(max=100)
DoRank — 相关性 + 热度 + 新鲜度 加权排序
Interpose — 黑名单过滤 + 低质过滤
Response — JSON + 前缀高亮 + 统计写入
📎

Hint 点后推荐（大家都在搜）

用户点击文档返回后，在点击的 Box 下方展示相关搜索词
CanSearch — doc_id 必填（用户刚点击的文档）
DoSearch — 标签匹配 + 行为共现 + 分类热词 + 查询扩展
DoRank — 共现强度 × 时效衰减 + 截断
Interpose — 过滤与当前 query 相同的词 + 去重
Response — 根据页面布局动态适配展示条数
🏠

Nav 教育页

搜索框下方热词，无需 query，个性化引导搜索
CanSearch — 允许空 query
DoSearch — 高频词 + 兴趣词 + 兜底词 混合召回
DoRank — 热度 × 新鲜度 × 个性化 + 去重
Response — JSON + 分类标签

搜索主流程 Pipeline

Template Method 8 阶段骨架，每个 Processor 由 YAML 配置驱动反射创建，按序执行。

BaseHandler::Search(session) — 8 阶段 Pipeline
0
InitSession
初始化 Session · 生成 TraceID · 设置超时 deadline
1
CanSearch
准入检查 + InterposeCheckQuery（query 级封禁词）
2
PreSearch — 查询理解
QueryParser::Parse() → 归一化 → 分词 → 同义词扩展 → 类别推断 → Embedding
3
DoSearch — 4 路召回
recall_pipeline.Execute()，每路独立 Processor，结果 append 到 recall_results
InvertedRecall (1000) VectorRecall (200) UserHistoryRecall HotContentRecall
4
DoRank — 粗排（3 打分器加权融合）
rank_pipeline.Execute() → sort by coarse_score → truncate top-500
BM25 (w=0.6) Quality (w=0.2) Freshness (w=0.2)
5
DoRerank — 精排
rerank_pipeline.Execute() → 10 维特征 → LightGBM / 内置决策树 → top-100
LGBMScorer (w=0.8)
6
DoInterpose — 过滤 + 后处理
filter_pipeline → postprocess_pipeline → final_results
Dedup (Jaccard≥0.9) Quality Spam Blacklist MMR (λ=0.7 → top-20)
7
SetResponse
分页截取 + JSON 序列化（doc_id, title, snippet, score, debug_scores）
8
ReportFinal
ScopeGuard 保证执行 · 各阶段耗时日志 · 干预记录上报

配置驱动

一个框架配置 + 每业务一个 Pipeline 配置，改配置不改代码。

config/framework.yaml
server:
  host: "0.0.0.0"
  port: 8080

embedding:
  provider: "onnx"
  dim: 768
  model_path: "models/bge-base-zh/model.onnx"

businesses:
  - business_type: "search"
    handler_name: "SearchBizHandler"
    session_name: "SearchSession"
  - business_type: "sug"
    handler_name: "SugBizHandler"
  - business_type: "hint"
    handler_name: "HintBizHandler"
  - business_type: "nav"
    handler_name: "NavBizHandler"

background:
  auto_train: { enable: true, interval_hours: 24 }
  auto_index_rebuild: { enable: true, interval_hours: 12 }
  sug_trie_rebuild: { enable: true, interval_sec: 3600 }
config/biz/search.yaml
recall_stages:
  - name: "InvertedRecallProcessor"
    params: { max_recall: 1000 }
  - name: "VectorRecallProcessor"
    params: { top_k: 200 }
  - name: "UserHistoryRecallProcessor"
    params: { max_recall: 50 }
  - name: "HotContentRecallProcessor"
    params: { max_recall: 30 }

coarse_rank_stages:
  - name: "BM25ScorerProcessor"
    params: { weight: 0.6, k1: 1.5, b: 0.75 }
  - name: "QualityScorerProcessor"
    params: { weight: 0.3 }
  - name: "FreshnessScorerProcessor"
    params: { weight: 0.1, decay_rate: 0.01 }

fine_rank_stages:
  - name: "LGBMScorerProcessor"

filter_stages:
  - name: "DedupFilterProcessor"
  - name: "QualityFilterProcessor"
  - name: "SpamFilterProcessor"
  - name: "BlacklistFilterProcessor"

postprocess_stages:
  - name: "MMRRerankProcessor"
    params: { lambda: 0.7, top_k: 20 }

技术栈

必需依赖精简，可选组件优雅降级。无 ONNX 时用伪向量，无 LightGBM 时用规则树，无 Faiss 时暴力搜索。

yaml-cpp配置解析必需
Protobuf数据序列化必需
jsoncppJSON API 响应必需
SQLite3文档 + 统计存储必需
ONNX RuntimeEmbedding 推理 (bge-base-zh)可选
LightGBM精排模型 (C API)可选
Faiss向量近似搜索 (HNSW)可选
spdlog结构化日志可选

API 接口

RESTful HTTP 接口，JSON 请求/响应。

接口方法说明关键参数
/api/v1/search POST 全文搜索 query, uid, page, page_size
/api/v1/sug GET 搜索建议（下拉联想） q（前缀）
/api/v1/hint GET 点后推荐（点击文档返回后展示「大家都在搜」） doc_id（必填）, query（可选）
/api/v1/nav GET 教育页（搜前热词引导） uid（可选，个性化）
/api/v1/doc/add POST 添加文档（同时建索引） doc_id, title, content, category
/api/v1/doc/update PUT 更新文档 doc_id + 更新字段
/api/v1/doc/delete DEL 删除文档 doc_id
/api/v1/event/click POST 上报点击事件（含特征快照） uid, doc_id, query, result_pos
/api/v1/event/like POST 上报点赞事件 uid, doc_id
/api/v1/admin/reload_model POST 精排模型热更新 model_path
/health GET 健康检查 + 索引统计 —
